# Story 1.4: Motor de Geração de Perfil

## Status
Draft

## Story

**As a** sistema,
**I want** processar as respostas do questionário,
**so that** gere um perfil cognitivo personalizado.

## Acceptance Criteria

1. Algoritmo processa respostas e gera perfil estruturado
2. Perfil contém: resumo (1-2 frases), insight central, sugestão prática
3. Perfil é salvo no banco de dados associado ao usuário
4. Lógica permite extensão futura (mais dimensões de perfil)
5. Testes unitários cobrem casos principais de geração

## Tasks / Subtasks

- [ ] **Criar Edge Function de Geração** (AC: 1, 2)
  - [ ] Criar `supabase/functions/generate-profile/index.ts`
  - [ ] Receber array de respostas
  - [ ] Implementar algoritmo de scoring
  - [ ] Gerar output estruturado

- [ ] **Algoritmo de Análise** (AC: 1, 4)
  - [ ] Calcular scores por categoria (0-100)
  - [ ] Identificar padrão de energia (manhã/tarde/noite)
  - [ ] Determinar estilo de execução (sequential/parallel/burst)
  - [ ] Calcular threshold de sobrecarga (1-10)

- [ ] **Geração de Textos** (AC: 2)
  - [ ] Templates de resumo por combinação de scores
  - [ ] Templates de insight por padrões detectados
  - [ ] Templates de sugestão prática
  - [ ] Opcional: usar LLM para personalização

- [ ] **Persistência do Perfil** (AC: 3)
  - [ ] Criar migration para tabela `profiles`
  - [ ] Salvar perfil com versão
  - [ ] Associar ao usuário (se logado) ou sessão

- [ ] **API Endpoint** (AC: 1)
  - [ ] POST `/functions/v1/generate-profile`
  - [ ] Request: `{ answers: DiscoveryAnswer[] }`
  - [ ] Response: `Profile`
  - [ ] Validação com Zod

- [ ] **Testes** (AC: 5)
  - [ ] Testes unitários do algoritmo de scoring
  - [ ] Testes de geração de texto
  - [ ] Testes de integração da edge function

## Dev Notes

### Tabela profiles (SQL)
```sql
CREATE TABLE profiles (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    session_id UUID, -- para usuários não logados
    summary TEXT NOT NULL,
    insight TEXT NOT NULL,
    suggestion TEXT NOT NULL,
    energy_pattern JSONB DEFAULT '{}',
    execution_style TEXT CHECK (execution_style IN ('sequential', 'parallel', 'burst')),
    overload_threshold INTEGER DEFAULT 5 CHECK (overload_threshold BETWEEN 1 AND 10),
    preferred_techniques TEXT[] DEFAULT '{}',
    scores JSONB DEFAULT '{}', -- scores por categoria
    version INTEGER DEFAULT 1,
    created_from TEXT DEFAULT 'discovery' CHECK (created_from IN ('discovery', 'adaptation', 'rediscovery')),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_profiles_user_id ON profiles(user_id);
CREATE INDEX idx_profiles_session_id ON profiles(session_id);
```
