# Story 1.1: Setup do Projeto e Infraestrutura Base

## Status
Draft

## Story

**As a** desenvolvedor,
**I want** ter o projeto configurado com estrutura, CI/CD e ambiente de desenvolvimento,
**so that** possamos começar a desenvolver features com qualidade.

## Acceptance Criteria

1. Repositório monorepo criado com estrutura de pastas definida (Turborepo)
2. Configuração de TypeScript 5.3+, ESLint e Prettier funcionando
3. Pipeline de CI básico rodando (lint, type-check) no GitHub Actions
4. Ambiente de desenvolvimento local documentado e funcional
5. README com instruções de setup
6. Estrutura de feature flags implementada (Flagsmith)
7. Projeto Supabase criado com schema inicial

## Tasks / Subtasks

- [ ] **Inicializar Monorepo com Turborepo** (AC: 1)
  - [ ] Criar estrutura base com `npx create-turbo@latest`
  - [ ] Configurar workspaces no `pnpm-workspace.yaml`
  - [ ] Criar pastas: `apps/mobile`, `apps/web`, `packages/shared`

- [ ] **Configurar TypeScript e Linting** (AC: 2)
  - [ ] Configurar `tsconfig.json` base em `packages/config`
  - [ ] Configurar ESLint com `@typescript-eslint`
  - [ ] Configurar Prettier com regras do projeto
  - [ ] Adicionar scripts no root `package.json`

- [ ] **Setup React Native** (AC: 1)
  - [ ] Inicializar projeto React Native 0.73 em `apps/mobile`
  - [ ] Configurar TypeScript no mobile
  - [ ] Instalar dependências base (React Navigation, Zustand)
  - [ ] Configurar iOS e Android builds básicos

- [ ] **Setup Next.js** (AC: 1)
  - [ ] Inicializar Next.js 14 em `apps/web`
  - [ ] Configurar App Router
  - [ ] Configurar Tailwind CSS

- [ ] **Configurar Supabase** (AC: 7)
  - [ ] Criar projeto no Supabase
  - [ ] Configurar Supabase CLI localmente
  - [ ] Criar migration inicial (tabelas: users, plans)
  - [ ] Configurar variáveis de ambiente

- [ ] **Configurar CI/CD** (AC: 3)
  - [ ] Criar `.github/workflows/ci.yml`
  - [ ] Jobs: lint, type-check, test (quando houver)
  - [ ] Configurar cache de dependências

- [ ] **Configurar Feature Flags** (AC: 6)
  - [ ] Criar conta Flagsmith (ou setup local)
  - [ ] Criar flags iniciais: `discovery_enabled`, `basic_plan_features`
  - [ ] Integrar SDK no mobile e web

- [ ] **Documentação** (AC: 4, 5)
  - [ ] Atualizar README com instruções de setup
  - [ ] Documentar variáveis de ambiente necessárias
  - [ ] Criar CONTRIBUTING.md básico

## Dev Notes

### Source Tree Esperada
```
nciaflux/
├── apps/
│   ├── mobile/          # React Native 0.73
│   └── web/             # Next.js 14
├── packages/
│   ├── shared/          # Types, schemas, utils
│   └── config/          # ESLint, TS configs
├── supabase/
│   ├── functions/       # Edge Functions
│   └── migrations/      # SQL migrations
├── turbo.json
├── package.json
└── pnpm-workspace.yaml
```

### Tech Stack (da Arquitetura)
- **Monorepo:** Turborepo 1.x + pnpm
- **Mobile:** React Native 0.73, TypeScript 5.3
- **Web:** Next.js 14, Tailwind CSS
- **Backend:** Supabase (PostgreSQL, Auth, Edge Functions)
- **Feature Flags:** Flagsmith

### Comandos Esperados
```bash
pnpm install          # Instalar dependências
pnpm dev              # Dev server (turbo)
pnpm lint             # Lint all packages
pnpm type-check       # TypeScript check
pnpm build            # Build all
```

### Testing

- **Framework:** Vitest (será configurado)
- **Localização:** `__tests__/` ao lado do código
- **Convenção:** `*.test.ts`
- **Nesta story:** Apenas estrutura, sem testes ainda
