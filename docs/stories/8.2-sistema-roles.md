# Story 8.2: Sistema de Roles no Time

## Status
Draft

## Story

**As a** gestor de time,
**I want** definir diferentes papéis para membros,
**so that** cada pessoa tenha permissões apropriadas.

## Acceptance Criteria

1. Roles: Owner, Admin, Manager, Member
2. Owner pode tudo + deletar time
3. Admin pode gerenciar membros e configs
4. Manager pode ver relatórios
5. Member só vê próprios dados + chat
6. UI clara mostrando role de cada um

## Dev Notes

```typescript
enum TeamRole {
  OWNER = 'owner',
  ADMIN = 'admin',
  MANAGER = 'manager',
  MEMBER = 'member',
}

const ROLE_PERMISSIONS: Record<TeamRole, string[]> = {
  owner: ['*'], // Tudo
  admin: ['manage_members', 'manage_settings', 'view_reports', 'manage_challenges'],
  manager: ['view_reports', 'view_members', 'create_challenges'],
  member: ['view_own', 'chat', 'participate_challenges'],
};
```

```sql
-- RLS Policy exemplo
CREATE POLICY "Managers can view team reports"
ON team_reports FOR SELECT
USING (
  EXISTS (
    SELECT 1 FROM team_members
    WHERE team_id = team_reports.team_id
    AND user_id = auth.uid()
    AND role IN ('owner', 'admin', 'manager')
  )
);
```
