# Story 8.6: Controle de Features por Gestor

## Status
Draft

## Story

**As a** gestor de time,
**I want** controlar quais features estão disponíveis para o time,
**so that** eu possa adaptar a experiência às necessidades do grupo.

## Acceptance Criteria

1. Ligar/desligar features específicas
2. Override da liberação progressiva
3. Features customizadas por membro
4. Bloquear features para simplificar
5. Preset de configurações (Iniciante, Padrão, Avançado)
6. Log de alterações

## Tasks / Subtasks

## Dev Notes

```typescript
interface TeamFeatureConfig {
  teamId: string;
  preset: 'beginner' | 'standard' | 'advanced' | 'custom';
  features: {
    [featureId: string]: {
      enabled: boolean;
      overrideLevel?: number;
    };
  };
  memberOverrides: {
    [userId: string]: {
      [featureId: string]: boolean;
    };
  };
  updatedAt: string;
  updatedBy: string;
}

const PRESETS = {
  beginner: {
    brain_dump: true,
    basic_tasks: true,
    crisis_mode: true,
    ai_chat: true,
    // Resto desabilitado
  },
  standard: {
    // Level 1-3 features
  },
  advanced: {
    // Todas features
  },
};
```

```sql
CREATE TABLE team_feature_config (
  team_id UUID REFERENCES teams PRIMARY KEY,
  preset TEXT DEFAULT 'standard',
  features JSONB DEFAULT '{}',
  member_overrides JSONB DEFAULT '{}',
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  updated_by UUID REFERENCES auth.users
);

CREATE TABLE team_feature_log (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  team_id UUID REFERENCES teams NOT NULL,
  changed_by UUID REFERENCES auth.users NOT NULL,
  change_type TEXT NOT NULL, -- 'preset', 'feature', 'member_override'
  old_value JSONB,
  new_value JSONB,
  reason TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
