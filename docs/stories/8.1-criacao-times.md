# Story 8.1: Criação e Gestão de Times

## Status
Draft

## Story

**As a** gestor/líder,
**I want** criar um time e convidar membros,
**so that** minha equipe possa usar o Mentes Brilhantes juntos.

## Acceptance Criteria

1. Criar time com nome e descrição
2. Gerar código/link de convite
3. Definir limite de membros (por plano)
4. Ver lista de membros do time
5. Remover membros
6. Transferir ownership do time
7. Deletar time (com confirmação)

## Dev Notes

### Schema Supabase

```sql
CREATE TABLE teams (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  owner_id UUID REFERENCES auth.users NOT NULL,
  plan TEXT DEFAULT 'pro',
  max_members INT DEFAULT 10,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE team_members (
  team_id UUID REFERENCES teams ON DELETE CASCADE,
  user_id UUID REFERENCES auth.users,
  role TEXT DEFAULT 'member', -- 'admin', 'manager', 'member'
  joined_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (team_id, user_id)
);

CREATE TABLE team_invites (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  team_id UUID REFERENCES teams ON DELETE CASCADE,
  code TEXT UNIQUE NOT NULL,
  created_by UUID REFERENCES auth.users,
  expires_at TIMESTAMPTZ,
  max_uses INT,
  uses INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- RLS
ALTER TABLE teams ENABLE ROW LEVEL SECURITY;
ALTER TABLE team_members ENABLE ROW LEVEL SECURITY;
ALTER TABLE team_invites ENABLE ROW LEVEL SECURITY;

-- Team members can view their team
CREATE POLICY "Members can view team" ON teams
  FOR SELECT USING (
    id IN (SELECT team_id FROM team_members WHERE user_id = auth.uid())
  );

-- Only admins can update team
CREATE POLICY "Admins can update team" ON teams
  FOR UPDATE USING (
    id IN (
      SELECT team_id FROM team_members
      WHERE user_id = auth.uid() AND role IN ('admin', 'owner')
    )
  );
```
