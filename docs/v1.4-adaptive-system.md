# v1.4 - Sistema Adaptativo para TDAH

> **Filosofia**: Não é o usuário que configura o app. É o app que se configura ao usuário.

## Visão Geral

O Mentes Brilhantes deve detectar padrões através da interação e ajustar automaticamente a experiência. Features são liberadas progressivamente baseado no perfil cognitivo e estabilidade do usuário.

```
┌─────────────────────────────────────────────────────────────┐
│              CICLO ADAPTATIVO                                │
│                                                              │
│   COLETA ──► DETECÇÃO ──► ADAPTAÇÃO ──► FEEDBACK            │
│      │                                      │                │
│      └──────────────────────────────────────┘                │
│                    (contínuo)                                │
└─────────────────────────────────────────────────────────────┘
```

## 2. Formas de Coleta de Dados

### 2.1 Coleta Ativa (usuário responde)

| Fonte | Dados Coletados | Frequência |
|-------|-----------------|------------|
| **Questionário Cronotipo** | Tipo (Leão/Urso/Lobo/Golfinho), horários ideais | Onboarding + opcional |
| **Questionário TDAH** | Subtipo (hiperativo/desatento/combinado), intensidade | Onboarding + opcional |
| **Inventário de Gatilhos** | Situações que causam overwhelm, paralisia | Onboarding + mensal |
| **Check-in Diário** | Humor (5 níveis), energia (1-5), sono | Diário |
| **Revisão Semanal** | Vitórias, desafios, aprendizados, nota da semana | Semanal |
| **Feedback de Adaptação** | "Isso ajudou?" após sugestões | Quando adapta |

### 2.2 Coleta Passiva (automática, sem fricção)

| Fonte | Dados Coletados | Como |
|-------|-----------------|------|
| **Horários de Uso** | Quando abre o app, duração das sessões | Timestamp de eventos |
| **Navegação** | Quais módulos usa mais, ordem de acesso | Page views |
| **Tarefas** | Criadas vs completadas vs abandonadas | Status changes |
| **Tempo por Tarefa** | Quanto demora em cada uma | Timer implícito |
| **Brain Dump** | Frequência, quantidade, horários de pico | Criação de itens |
| **Modo Foco** | Duração real vs planejada, interrupções | Timer + eventos |
| **Modo Crise** | Frequência, horários, duração | Ativações |
| **Chat IA** | Temas recorrentes, pedidos de ajuda, tom | NLP nos prompts |
| **Calendário** | Eventos criados vs comparecidos | Status de eventos |
| **Rotinas** | Taxa de conclusão, passos pulados | Checkboxes |

### 2.3 Coleta via Chat IA

A IA pode fazer perguntas naturais durante conversa:

```
Usuário: "Tô travado, não consigo começar nada"

IA: "Entendo... isso tá acontecendo mais de manhã
     ou de tarde geralmente?"

Usuário: "Sempre depois do almoço"

IA: [Registra: gatilho = pós-almoço, tipo = paralisia]
    "Faz sentido, muita gente com TDAH tem uma
     queda de energia nesse horário. Quer que eu
     te sugira tarefas mais leves pra esse período?"
```

### 2.4 Coleta por Inferência

| Observação | Inferência |
|------------|------------|
| App aberto 3am várias vezes | Possível insônia ou cronotipo noturno |
| Muitas tarefas criadas, poucas completas | Overwhelm ou planejamento irreal |
| Brain dump explosivo às segundas | Acúmulo de fim de semana |
| Modo crise sempre às 22h | Gatilho noturno (cansaço + reflexão) |
| Tarefas completas só com timer | Precisa de estrutura externa |
| Abandona tarefas longas | Melhor quebrar em subtarefas |

## 3. Sistema de Detecção de Padrões

### 3.1 Padrões Monitorados

```typescript
interface PatternDetection {
  // Energia
  energyPeaks: TimeRange[];        // Horários de alta energia
  energyDips: TimeRange[];         // Horários de baixa

  // Produtividade
  productiveDays: DayOfWeek[];     // Dias mais produtivos
  difficultDays: DayOfWeek[];      // Dias difíceis
  taskCompletionRate: number;      // % de conclusão
  averageTaskDuration: number;     // Tempo médio por tarefa

  // Overwhelm
  overwhelmTriggers: Trigger[];    // O que causa
  overwhelmTimes: TimeRange[];     // Quando acontece
  crisisModeFrequency: number;     // Frequência modo crise

  // Comportamento
  appUsageTimes: TimeRange[];      // Quando usa o app
  preferredModules: Module[];      // O que mais usa
  avoidedModules: Module[];        // O que evita

  // Social (quando habilitado)
  socialComfort: 'low' | 'medium' | 'high';
  responseLatency: number;         // Tempo para responder
}
```

### 3.2 Algoritmo de Estabilidade

```typescript
function calculateStabilityScore(user: User): number {
  const factors = {
    checkInConsistency: 0.25,    // Faz check-in regularmente?
    moodVariance: 0.20,          // Humor estável?
    taskCompletion: 0.20,        // Completa o que planeja?
    crisisFrequency: 0.20,       // Pouco modo crise?
    routineAdherence: 0.15,      // Segue rotinas?
  };

  // Score 0-100
  // < 30: Instável (modo proteção)
  // 30-60: Moderado (features básicas)
  // 60-80: Estável (libera mais features)
  // > 80: Muito estável (todas features disponíveis)
}
```

## 4. Sistema de Adaptação

### 4.1 Adaptações Automáticas

| Detecta | Adapta |
|---------|--------|
| Energia baixa (check-in) | Mostra só 1-2 tarefas, esconde resto |
| Overwhelm frequente | Reduz opções visíveis, simplifica UI |
| Horário de baixa | Sugere pausa, tarefas leves |
| Horário de pico | Sugere tarefa importante, modo foco |
| Muitas tarefas abertas | "Quer focar em só 3 hoje?" |
| Modo crise recente | Modo gentil por 24h |
| Padrão de insônia | Sugere rotina noturna, reduz notificações |
| Boa sequência | Celebra, sugere desafio novo |

### 4.2 Liberação Progressiva de Features

```
NÍVEL 1 - Todos (início)
├── Brain Dump
├── Tarefas básicas (max 5 visíveis)
├── Check-in
├── Modo Crise
└── Chat IA (apoio)

NÍVEL 2 - Após Perfil (cronotipo + TDAH completos)
├── Planner com horários sugeridos
├── Rotinas personalizadas
├── Projetos
└── Modo Foco

NÍVEL 3 - Estabilidade > 50 (2+ semanas consistente)
├── Todas tarefas visíveis
├── Relatório Pessoal
├── Revisão Semanal
└── Calendário completo

NÍVEL 4 - Estabilidade > 70 + Plano Plus
├── Multi-device sync
├── Parceiro accountability (1:1)
└── Comunidade (leitura)

NÍVEL 5 - Liberado por Gestor/Terapeuta
├── Chat em grupo
├── Features avançadas
└── Integrações externas
```

## 7. Gamificação para TDAH

### 7.1 Princípios

> **Gamificação TDAH ≠ Gamificação tradicional**

| Tradicional | TDAH-friendly |
|-------------|---------------|
| Punição por falha | Sem punição, só celebração |
| Streaks rígidos | Streaks flexíveis ("quase lá!") |
| Rankings competitivos | Progresso pessoal |
| Recompensas distantes | Dopamina imediata |
| Metas grandes | Micro-vitórias |

### 7.2 Níveis de Gamificação

```
┌─────────────────────────────────────────────────────────────┐
│              GAMIFICAÇÃO PROGRESSIVA                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  NÍVEL 1 - BÁSICO (todos, desde o início)                   │
│  ├── 🎉 Celebrações ao completar tarefa                     │
│  ├── 💬 Mensagens de encorajamento                          │
│  ├── ✨ Animações suaves de progresso                       │
│  └── 🌟 "Vitória do dia" no dashboard                       │
│                                                              │
│  NÍVEL 2 - INTERMEDIÁRIO (estabilidade > 40%)               │
│  ├── 🔥 Streaks gentis (sem zerar, só "pausa")              │
│  ├── 🏆 Conquistas simples (primeira tarefa, 7 dias, etc.)  │
│  ├── 📊 XP e nível pessoal                                  │
│  └── 🎁 Recompensas visuais (temas, avatares)               │
│                                                              │
│  NÍVEL 3 - AVANÇADO (estabilidade > 70% + opt-in)           │
│  ├── 🎯 Desafios semanais opcionais                         │
│  ├── 👥 Conquistas de time (se em grupo)                    │
│  ├── 📈 Rankings amigáveis (só se quiser)                   │
│  └── 🏅 Badges especiais                                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 Exemplos de Conquistas

| Conquista | Descrição | XP |
|-----------|-----------|-----|
| **Primeiro Passo** | Completou primeira tarefa | 10 |
| **Brain Dumper** | Esvaziou a mente 10 vezes | 25 |
| **Madrugador/Coruja** | 7 dias seguindo seu cronotipo | 50 |
| **Modo Zen** | Usou modo crise e se recuperou | 30 |
| **Sequência Suave** | 3 dias com check-in | 20 |
| **Explorador** | Testou todos os módulos | 40 |
| **Auto-conhecimento** | Completou perfil TDAH | 35 |

## 10. Métricas de Sucesso

| Métrica | Meta |
|---------|------|
| Check-in diário | > 60% dos dias ativos |
| Retenção 30 dias | > 40% |
| Modo crise reduzido | -20% após 1 mês |
| Tarefas completadas | +15% após adaptação |
| NPS | > 50 |
| "App me entende" | > 70% concordam |

---

> **Lembrete**: Cada feature deve passar pelo filtro:
> "Isso ajuda alguém com TDAH a ter um dia melhor, ou só adiciona complexidade?"
